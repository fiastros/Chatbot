## Detailed Component Descriptions

ROOT FILES
-----------------------------------------
File: .env
Libraries: None
Task: Stores sensitive API keys for external services like Neo4j AuraDB, Groq API, and LangSmith.

File: pyproject.toml
Libraries: uv (Package Manager)
Task: Replaces requirements.txt to define Python versions and dependencies cleanly.

File: uv.lock
Libraries: uv (Package Manager)
Task: Auto-generated by `uv` to lock exact dependency versions for reproducible builds.

File: main.py
Libraries: None
Task: The primary entry point. Intercepts the user's prompt, checks for safety, and triggers the orchestrator.


PHASE 1: src/ingestion/ (Offline Data Pipeline)
-----------------------------------------
File: parser.py
Libraries: docling
Task: Extracts and parses PDFs, Word docs, and images directly into hyper-clean Markdown.

File: chunker.py
Libraries: Built-in Python string/regex libraries
Task: Implements semantic and hierarchical chunking, uses sliding windows, deduplicates data, enriches metadata, and attaches File URIs.

File: embeddings.py
Libraries: sentence-transformers, torch
Task: Generates mathematical vector embeddings for text and images using Apple Metal Performance Shaders (MPS).

File: databases.py
Libraries: chromadb, Neo4j AuraDB, sqlite3
Task: Manages parallel storage across multimodal vectors, the knowledge graph, and untouched raw files.


PHASE 2: src/orchestration/ (Initiation & Orchestration)
-----------------------------------------
File: manager.py
Libraries: langgraph
Task: Determines workflow/execution patterns, analyzes intent, breaks complex queries into sub-tasks, and delegates them via state-machine workflows.

File: cache.py
Libraries: gptcache
Task: Checks for recently asked exact questions and serves cached answers instantly to bypass the pipeline.

File: llm_config.py
Libraries: Ollama, LM Studio, Groq API
Task: Configures local SLMs (Phi-3-Mini, Moondream2) and manages cloud API connections for logic.


PHASE 3: src/agents/ (Agentic Execution)
-----------------------------------------
File: researcher.py
Libraries: langchain, langchain-core
Task: Uses Reasoning/Planning/Acting (ReAct/CoT) to execute the Orchestrator's plan.

File: tools.py
Libraries: DuckDuckGoSearchRun
Task: Defines external tools, like scraping live news and web data when necessary.


PHASE 4: src/rag_pipeline/ (Retrieval & Search Tool)
-----------------------------------------
File: router.py
Libraries: semantic-router
Task: A lightweight classifier that routes sub-queries to the most efficient database or bypasses LLMs entirely for simple queries.

File: retriever.py
Libraries: LangChain Retriever core
Task: Handles query transformation (e.g., HyDE), fetches data using hybrid search (dense + sparse), and extracts relevant sentences via contextual compression.

File: reranker.py
Libraries: flashrank
Task: Strictly re-orders newly retrieved chunks based on absolute relevance optimally on CPU/Apple Silicon.


PHASE 5: src/augmentation/ (Context Augmentation)
-----------------------------------------
File: masking.py
Libraries: presidio-analyzer, presidio-anonymizer
Task: Strips sensitive PII from the text before it enters the prompt.

File: prompts.py
Libraries: LangChain Prompts
Task: Injects historical Q&A examples automatically (dynamic few-shot prompting) and manages context window token limits.


PHASE 6: src/synthesis_output/ (Validation & Synthesis)
-----------------------------------------
File: schemas.py
Libraries: pydantic
Task: Forces generated answers into strict, validated JSON schemas.

File: synthesizer.py
Libraries: instructor
Task: Patches LLM APIs for Pydantic outputs, constructs final responses with clickable citations, and handles Critic Agent self-correction loops and HITL pauses.

File: guardrails.py
Libraries: nemoguardrails
Task: Checks final outputs to ensure no hallucinations or toxic content exist before showing the user.


PHASE 7: src/llmops/ (Security & Observability)
-----------------------------------------
File: telemetry.py
Libraries: langsmith
Task: Visualizes step-by-step agent thoughts, logs latency metrics, and routes percentage traffic for background shadow testing.

File: evaluation.py
Libraries: ragas
Task: Automates quality assurance by grading outputs based on Context Precision, Answer Relevance, and Faithfulness.


TESTING SUITE: tests/ (Traditional Software Testing)
-----------------------------------------
File: conftest.py
Libraries: pytest 
Task: Provides shared testing fixtures across the entire suite, such as mock database connections, mocked API keys, or simulated LLM responses.

Folder: unit/ (test_parser.py, test_chunker.py, test_masking.py)
Libraries: pytest, unittest.mock
Task: Tests isolated functions and components independently without calling external APIs or loading massive LLMs (e.g., verifying the chunker splits a string correctly).

Folder: integration/ (test_databases.py, test_tools.py, test_rag_flow.py)
Libraries: pytest
Task: Tests multiple components working together, ensuring data flows correctly from your retrieval logic into your local database mockups.

Folder: e2e/ (test_full_pipeline.py)
Libraries: pytest
Task: End-to-End regression tests simulating a full user interaction. It sends a prompt into the main application logic and verifies that a complete, structurally sound final output is generated.
